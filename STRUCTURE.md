## ğŸ¨ Design Patterns

### 1. Barrel Pattern
Each directory has an `index.ts` that re-exports:
```typescript
// src/types/index.ts
export type { FileSizeUnit, NormalizedFileSize } from './common.types';
export type { FormErrors, ValidationResult } from './form.types';
```

### 2. Separation of Concerns
- **Types**: No implementation, only type definitions
- **Utils**: Pure functions, no side effects
- **Constants**: Immutable configuration
- **Modules**: Public API composition

### 3. Discriminated Unions
Type-safe conditional handling:
```typescript
type ValidationResult<T> =
  | { success: true; data: T }
  | { success: false; errors: FormErrors };
```

### 4. Immutability
- `readonly` modifiers
- `as const` assertions
- `Object.freeze()` calls
